# AuthService - Rama Developer

Esta rama está destinada al desarrollo activo del microservicio de autenticación. Aquí se integran nuevas funcionalidades, correcciones y mejoras antes de ser promovidas a la rama principal (`main`).

## Buenas prácticas
- Realiza commits frecuentes y descriptivos.
- Sincroniza regularmente con `main` para evitar conflictos.
- Ejecuta pruebas antes de subir cambios.

## Flujo de trabajo recomendado
1. Crea una rama `feature/` para cada nueva funcionalidad.
2. Realiza tus cambios y pruebas en la rama de feature.
3. Haz merge de tu feature a `developer` mediante Pull Request.
4. Una vez estable, se promueve a `main`.

## Documentación
Consulta la documentación en la carpeta `docs/` (local, no en el repo) y los archivos de especificación en la raíz del proyecto.
